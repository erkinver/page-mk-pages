# page-mk-pages

## 项目简介

`page-mk-pages` 是一个用于归档和发布独立 HTML 页面的仓库。它不针对任何特定域名或网站，而是作为一个通用的静态页面集合，方便管理和展示各类独立页面内容。

## 目录结构

```
.
├── pages/            # 存放所有独立 HTML 页面文件
├── assets/           # 公共资源文件（CSS、JS、图片等）
├── index.html        # 仓库首页（可选）
└── README.md         # 本文件
```

- `pages/`：每个子目录或文件代表一个独立的归档页面，命名清晰便于识别。
- `assets/`：存放页面间共享的样式、脚本或图片资源。
- `index.html`：可选的入口页面，用于导航或展示页面列表。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，可直接在浏览器中打开浏览。
- 页面内容不依赖外部服务，所有资源（样式、脚本、图片）尽量本地化或通过公共 CDN 引用。
- 归档页面可能包含说明文字、功能演示、工具页面或静态信息展示等，用途灵活。
- 每个页面的文件名应简洁明了，反映其内容或功能。

## 维护说明

- 如需添加新页面，请在 `pages/` 下创建新的 HTML 文件，并确保其自包含。
- 更新或修改已有页面时，请保持文件结构与命名一致。
- 公共资源如有更新，请同步至 `assets/` 目录，并检查引用路径是否正确。
- 本仓库不涉及任何商业推广、域名绑定或流量引导，仅作为页面归档与发布用途。

## 使用方式

1. 克隆本仓库到本地。
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件查看。
3. 也可将仓库部署到静态托管服务（如 GitHub Pages），通过仓库地址访问页面。
